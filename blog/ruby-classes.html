Ruby Classes

March 15, 2016

Today, we are going to touch ruby classes. What they? Well, they are like blueprints for making objects, a sort of cookie cutter if I may say. In your code you define classes in the intent to create objects that have the same characteristics. For example; to be able to illustrate that let’s refer on real world object which will be the Army here that recruits and trains individual to be come soldiers which are the objects in this case. First let’s define our class:

	class Army 
	end

No we have defined our Army class, it’s time to build up, and we think in terms of the objects, what are the characteristics of a soldier, they have a name of course, age, a rank to start and many others things, but in this blog, we will just use name and rank, now we know the details about our soldier object, we can initialize those information with the initialize method:

	class Army 
	  
	  def initialize(name,rank)
	
	end

Let’s keep on, now we have values, we need variables to hold our name and rank values, but what type of variables? Instance variables, they are one of the best features in ruby classes. One great benefit of instance variables is, unlike local variables they can be used in any method within a class, they are preceded with the @ sign. Let’s put them to work:

	class Army 
		
	  def initialize(name,rank)
	  @name = name
	  @rank = rank
	end

Our class is fully initialized with all the details we need for the soldier object, for the program to be able to read these information we have to define methods to do that:

	class Army 
		
	  def initialize(name,rank)
		@name = name
		@rank = rank
	  end

	  def name
		puts @name
	  end

	  def rank
		puts @rank
	  end
	end

Awesome, the program can read now, note that there other ways of doing this and we can also use a method that update the values. But we’ll just stick to a reading method here. Now it's time to recruit and train a soldier:

Soldier = Army.new(“Fred”,8)

The line above creates a new soldier objects with name and rank. Let’s print the name of the soldier

member.name #=> Fred

Fred is in the army now for quite some time, he even did a tour in Iraq; Let’s check if his rank has changed, we will define an instance method to do so, an instance method is any method that we can call on an object. So our reader method is also an instance method:

	class Army 
	  
	  def initialize(name,rank)
		@name = name
		@rank = rank
	end

	  def name
		puts @name
	  end

	  def rank
		puts @rank
	  end

	  def status
		if @rank < 10 
			puts "Recruit"
		else
			puts "Captain"
	    end
	  end
	end

Soldier = Army.new(“Fred”,8)
member.name #=> Fred
soldier.status #=> Recruit

We used the instance method on the soldier object to check his rank; despite a tour in Iraq Fred is still a recruit.
