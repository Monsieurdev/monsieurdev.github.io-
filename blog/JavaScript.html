Ruby Hashes vs. Javacript Objects

April 8, 2016

If you are a Ruby programmer, and decide to start learning JavaScript, you’ll see a lot of similarities, among them Objects and hashes are great examples to showcase these similarities. However they do differ in many ways too. So without further due let’s illustrate.

First thing first off, both Hashes and Objects are created using curly braces:

hash ={}            object = {};

In the example above, both the hash and the Object are empty. In Ruby, hashes are composed of key value pairs, they are two main notations for key/value pairs, we have the rocket notation “=>” and the symbol notation with a colon “:” behold: 

hash = {
	'name'=> 'John',
	:job => 'lawyer'
}

We also have key/value situation with Objects, but they are just referred as properties, so Objects contain properties and their values. Unlike hashes, Objects don’t have the rocket notation; they only operate with the colon notation, and the properties cannot be symbols like in Ruby:

object = {
	name : 'John',
	job : 'lawyer'
};

With hashes, you can only access keys using the bracket notation:

	hash = {
	'name'=> 'John',
	:job => 'lawyer'
}

puts hash['name']  #=> John

JavaScript gives two options, you could either use the bracket notation or the dot notation. Isn’t that awesome?

	object = {
	name : 'John',
	job : 'lawyer'
};

console.log(object.name); #=> john
console.log(object['job']); #=> lawyer

Another similarity is, the nested principle applies to both. You can have hashes inside a hash as you can have objects inside an Object, you can also have arrays inside them: 

hash = {                                     
	'name'=> 'john',
	:job => 'lawyer',
	'wife' => {'name' => 'Lisa'},            
	'kids' => ['Mary', 'Steve']
}

object = {
	name : 'john',
	job : 'lawyer',
	wife : {'name' : 'Lisa'},
	kids : ['Mary', 'Steve']
};
